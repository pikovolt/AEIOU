## A. 切り貼りと中抜き：フレーム補正の仕様

### A.1 データ構造

中抜き範囲・切り貼り範囲はそれぞれ `Range` クラスのリストとして保持される。

```
Range {
    Top: int      // 開始行（0始まり）
    Bottom: int   // 終了行（0始まり、inclusive）
}
```

- `delRange`: List<Range> — 中抜き範囲
- `addRange`: List<Range> — 切り貼り範囲

1つのリストに複数の範囲を登録可能。範囲同士の重複チェックは切り貼り設定時にのみ行う。

### A.2 中抜き範囲の操作

**設定**: 現在の選択範囲を `Range(top, bottom)` として `delRange` に追加する。

**解除**: 選択範囲の先頭行が含まれる `Range` を `delRange` から削除する。

**AEコピー時の効果**: 中抜き範囲内の行はスキップされ、出力フレーム番号は中抜き行数分だけ減算される。

```
出力フレーム番号 = 行番号 - (その行より手前にある中抜き行の合計数)
```

### A.3 切り貼り範囲の操作

**設定**:
1. 選択範囲と既存の切り貼り範囲が重なる場合、重なる範囲を先に削除する（行の詰め＋中抜き/切り貼り範囲の再計算を伴う）
2. 選択範囲の位置に空白行を挿入する（`insertToAllCell`）
3. 中抜き/切り貼り範囲を再計算する
4. 新しい範囲を `addRange` に追加する

**解除**:
1. 選択範囲の先頭行が含まれる `Range` を特定する
2. その範囲の行を削除する（`cutToAllCell`）
3. 中抜き/切り貼り範囲を再計算する
4. `addRange` からエントリを削除する

### A.4 範囲の再計算（挿入/削除時）

行の挿入・削除が発生した場合、中抜き/切り貼り範囲のTop/Bottomを補正する。中抜き・切り貼りとも同一のロジック。

```
挿入時:
    各範囲について、Range.Top >= 挿入位置 の場合:
        Range.Top  += 挿入行数
        Range.Bottom += 挿入行数

削除時:
    各範囲について、Range.Top >= 削除位置 の場合:
        Range.Top  -= 削除行数
        Range.Bottom -= 削除行数
```

挿入位置より手前の範囲は影響を受けない。

### A.5 中抜きを考慮したカーソル移動量の計算

Enter / + / - / . キー押下時、選択範囲の高さ分だけ下に移動するが、中抜き範囲内の行は「移動したうちに数えない」。

```
入力:
    cur   = 選択範囲の先頭行
    range = 選択範囲の高さ（移動したいコマ数）

処理:
    line   = 1（実際に何行下に進んだか）
    length = 0（中抜きを除いた有効な移動量）

    while length < range:
        if line > 総行数: return -1（範囲外）
        if (cur + line) がいずれかの中抜き範囲内:
            （何もしない）
        else:
            length++
        line++

    return line - 1
```

### A.6 フレーム表示文字列の生成

行番号から「シート番号/コマ番号」形式の表示文字列を生成する。切り貼り範囲を考慮する。

```
入力: frm（行番号、0始まり）

1. 切り貼り範囲の総コマ数を積算（addFrame）
2. 現在行が切り貼り範囲内かを判定

切り貼り範囲内の場合:
    ページ番号 = 0（固定）
    コマ番号 = (範囲内ローカル位置) % (シート秒数 × fps)

切り貼り範囲外の場合:
    補正フレーム = frm - addFrame + 1
    ページ番号 = (補正フレーム - 1) / (シート秒数 × fps) + 1
    コマ番号 = (補正フレーム - 1) % (シート秒数 × fps)

表示形式: "ページ/コマ"（各2桁ゼロ埋め）
ページ表示は 12コマ毎（24fps時）または 15コマ毎（30fps時）のみ。それ以外の行は空白5文字。
```

## B. Undo の仕様

### B.1 データ構造

```
UndoOperation {
    Column: int       // 対象セルの列
    Row: int          // 対象セルの行
    OldValue: string  // 変更前の値
    NewValue: string  // 変更後の値
}

UndoGroup {
    Id: int                             // 連番ID
    Title: string                       // メニュー表示用（"入力", "削除", "ペースト" 等）
    Target: UndoOperationTarget         // 操作対象の分類
    Rect: Rect                          // コピー/ペースト時の範囲記録
    Operations: List<UndoOperation>     // 操作のリスト
}
```

### B.2 操作対象の分類（UndoOperationTarget）

| 値 | 意味 | Undo時の復元先 |
|----|------|---------------|
| ToGrid | グリッドへの編集 | グリッドに OldValue を書き戻す |
| ToCopyBuffer | コピーバッファへの書き込み | バッファに OldValue を書き戻す |
| MoveToCopyBuffer | グリッド→バッファへ移動（カット） | グリッドに NewValue を書き戻し、バッファに OldValue を書き戻す |
| FromCopyBuffer | バッファ→グリッドへ（ペースト） | グリッドに OldValue を書き戻し、コピー範囲を復元する |

### B.3 Undo実行の手順

1. リストの末尾から `UndoGroup` を1つ取り出す（リストから削除）
2. `Operations` を逆順に走査し、Target に応じた復元処理を行う
3. セルの空白⇔入力済み状態が変わった場合は使用状況カウントを修正する
4. メニュー表示を更新する

### B.4 履歴の上限管理

`UndoOperation` の合計数が上限（設定値）を超えた場合、古い `UndoGroup` から順に削除する。判定は新しい方から積算し、上限を超えた時点でそれより古いグループをすべて削除する。

1回の操作で上限を超える `UndoGroup` は履歴に残らない。

### B.5 Undoが無効化される操作

以下の操作は Undo 履歴をフラッシュ（全消去）する:

- 行の挿入（Insert）
- 行の削除（Shift+Delete）
- セル枚数の変更
- セルの挿入/削除
- 切り貼り範囲の設定/解除
- AEからのペースト
- STS読み込み

理由: 行数・列数が変わると、履歴内の行列インデックスが無効になるため。

## C. 自動数値入力（+/-）の仕様

### C.1 + キー（カウントアップ入力）

1. カーソル位置の手前（上方向）を走査し、最初に見つかった数値を取得する
2. 空白行はスキップする
3. カラセルを見つけた場合は処理を中断する（何も入力しない）
4. 数値が見つかった場合、その値 + 1 をカーソル位置に入力する
5. 数値が見つからなかった場合、1 を入力する
6. 入力後、中抜きを考慮したカーソル移動（A.5節）で次の行に進む

### C.2 - キー（カウントダウン入力）

+ キーと同一の走査ロジック。以下が異なる:

- 見つかった数値 - 1 を入力する
- 結果が 1 未満にならないよう下限を 1 に制限する
- 数値が見つからなかった場合、0 を入力する

## D. カラセルと継続線の仕様

### D.1 カラセルの内部表現

カラセルは文字列として保持される。デフォルト値は設定ファイルで指定可能（`karaCell` タグ）。`.`（ドット）キーで入力する。

AEコピー時にはカラセル行はスキップされる（出力に含まれない）。

### D.2 継続線の判定

セルが空白の場合に、そのセルより上方向を走査して以下を判定する:

- 空白: スキップして走査を続行
- カラセル: 継続なし（false）
- 数値入力あり: 継続あり（true）
- 先頭行に達しても入力が見つからない: 継続なし（false）

継続ありの場合、セル描画時に縦棒線を描画する。

## E. AEからのペースト処理

### E.1 パース手順

クリップボードのテキストを改行で分割し、以下の順に処理する:

1. **ヘッダ検証**: 先頭行に `"Adobe After Effects "` と `"Keyframe Data"` の両方が含まれることを確認
2. **FPS取得**: 3行目（0始まり: index 2）の `"Units Per Second"` 行からタブ区切りで3番目の値を取得し、設定に反映する。メニューのチェック状態も追従する
3. **Time Remap行の検索**: `"Time Remap"` を含む行を前方検索する
4. **キーフレーム読み取り**: Time Remap行 + 2行目以降を、タブ区切りが1要素以下の行が来るまで読み取る

### E.2 値の変換

```
各行: \t{frame}\t{seconds}

frame = int(タブ区切り2番目)
seconds = double(タブ区切り3番目)
セル値 = Round(fps × seconds) + 開始フレーム番号
```

`Round` は四捨五入（`Math.Round`）。書き込み先はカーソル位置の列。

## F. STS形式のバイナリ仕様

### F.1 ファイル構造

| オフセット | サイズ | 内容 |
|-----------|--------|------|
| 0 | 18 bytes | ヘッダ: `0x11` + `"ShiraheiTimeSheet"` |
| 18 | 1 byte | 列数（uint8） |
| 19 | 4 bytes | 行数（uint32, little-endian） |
| 23 | 列数 × 行数 × 2 bytes | セルデータ |
| 続き | 可変長 | セル名称データ |

### F.2 セルデータ

各セルは uint16（2 bytes, little-endian）。列単位で行方向に連続して格納される。

保存時: 各行のセル値を書き出す。空白セルは直前の値を継続する（タイムシートの継続を再現）。

読込時: 値が変化した行のみをグリッドに反映する（値が変わらない行は空白のまま＝継続を意味する）。

### F.3 セル名称データ

列ごとに以下の形式:

| サイズ | 内容 |
|--------|------|
| 1 byte | 名称のバイト長（Shift_JIS） |
| n bytes | 名称文字列（Shift_JIS） |

## G. 入力補助機能の仕様

### G.1 連番作成

パラメータ: 開始番号、ステップ数、番号スキップ（チェックボックス）

```
入力範囲: 選択範囲の先頭列、先頭行から選択高さ分
値: 開始番号から、ステップ数間隔で入力

番号スキップ OFF: 行をステップ数ずつ進み、値は1ずつ増加
番号スキップ ON:  行をステップ数ずつ進み、値もステップ数ずつ増加
```

### G.2 繰り返し

パラメータ: 開始番号、終了番号、コマ打ち数、繰り返し数、スキップ数、挿入番号（任意）

```
基本動作:
    開始番号から終了番号までを、コマ打ち数の間隔で入力する
    それを繰り返し数の回数だけ繰り返す

スキップ数 > 0:
    スキップ数 + 1 行に1回だけ入力する（入力回数が減る）

挿入番号あり:
    通常の番号と挿入番号を交互に入力する
    （例: 1, 挿入, 2, 挿入, 3, 挿入, ...）
```

### G.3 置換

パラメータ: 置換前の値、置換後の値

選択範囲の先頭列を走査し、置換前の値と完全一致するセルを置換後の値に書き換える。部分一致ではない。

### G.4 反転

選択範囲の先頭列内で、値が入っているセルの順序を逆転する。空白行の位置は維持される。

```
例:
    入力前: 1, "", 3, "", 5
    反転後: 5, "", 3, "", 1
```

### G.5 四則演算

パラメータ: 演算式（先頭1文字が `+` `-` `*` `/`、以降が数値）

選択範囲内の全セル（空白・カラセルを除く）に対して、指定の演算を適用する。乗除算は元の値が 0 の場合はスキップする。

変換に失敗した場合は途中までの操作を Undo で巻き戻す。

### G.6 複製

選択範囲の先頭列の内容を、選択範囲の直下に1回コピーする（2回繰り返し固定）。コピー後、選択範囲はコピー先に移動する。

## I. その他の実装仕様

### I.1 セル使用状況カウント

列ごとに `aryCellUsedCount[col]` で入力済みセル数を管理する。空白→入力時に +1、入力→空白時に -1。描画時に列の背景色を「未入力/入力済」で切り替えるために使用する。

### I.2 基準線の描画判定

行番号から切り貼りフレーム数を差し引いた補正値に対して、以下の判定を行う:

```
24fps時:
    SheetDivide(12等) コマ毎 → EveryNFrames（細い基準線）
    24コマ毎          → EverySec（1秒線）
    SheetSec×24 コマ毎 → EverySheet（シート線）

30fps時:
    15コマ毎          → EveryNFrames
    30コマ毎          → EverySec
    SheetSec×30 コマ毎 → EverySheet
```

### I.3 クリップボード操作のリトライ

`SetClipboardTextWithRetry` により、最大5回・100ms間隔でリトライする。他アプリがクリップボードをロックしている場合への対策。最終リトライで失敗した場合は例外をスローする。

### I.4 JS経由のAEコピー（Script仲介）

通常のKeyframe Dataフォーマットではなく、JSON風の文字列を生成してクリップボードにコピーし、AfterFX.exe を `-r` オプション付きで起動して jsx スクリプトを実行する。

```
出力形式:
{property:'Time Remap',scale:24.00,keys:[{'t':0,'v':[0]},{'t':3,'v':[2]}, ...]}
```

通常コピーとの違い: 秒数計算時に fps で割らない（スクリプト側に委譲）。

### I.5 画面2/3スクロール

カーソル位置が表示領域の下から2/3を超えた場合、超過分だけ表示をスクロールする。Enter, ↓, ., +, -, K キーの処理後に呼ばれる。

```
border = 表示行数 × 2/3
currentRow = カーソル行 - 表示先頭行
if currentRow > border:
    forwardCount = currentRow - border
    表示先頭行 += forwardCount（ただし末端を超えない）
```
